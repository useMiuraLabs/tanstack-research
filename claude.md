# tanstack-research

## このリポジトリの役割

TanStack エコシステム (Router / Start / Query など) を学ぶための学習用リポジトリ。
各ライブラリごとにサブディレクトリを切り、独立した検証環境として扱う。

- `router/` — TanStack Router の学習 (詳細は `router/claude.md`)
- 今後追加予定: `start/`, `query/` など

## ユーザーが大切にしていること

### 1. FW / library の設計思想に則ることが一番大事
- 「動けばいい」ではなく、その FW / library が想定する使い方・設計哲学に沿って書く
- 自己流の回避策やアンチパターンを避け、公式が推奨するパターンを採用する
- 思想に反する実装を提案する前に「なぜこの FW はこう設計されているか」を考える

### 2. 情報は一次情報を必ず添えること
- 回答や提案には公式ドキュメント・公式 GitHub・公式ブログなどの一次情報の URL を必ず添える
- 二次情報 (Qiita / Zenn / 個人ブログ等) のみを根拠にしない
- Claude の内部知識だけで断定せず、最新の一次情報を WebFetch などで確認してから答える
- 一次情報が確認できない場合は「未確認」である旨を明示する

### 3. 対比で説明
- 解説や質問は Next.js ではこうだが， xxx ではこう と対比で説明してほしい

## 進め方の原則

- 基本的に手を動かすのはAI => AIが公式通りに作ったものをユーザーが参照し，質問は反証，実験を行う
- 各ライブラリの学習方針はサブディレクトリの `claude.md` に記載する

## 必ず守ること
- 一次情報の参照と説明 + 提供
- 敬語を使用
- 誤った場合は謝罪を必ずする

### 無駄な提案の禁止
無駄な提案はユーザーの判断を鈍らせるだけではなくストレスを溜めている

#### NG例

1. claude「xxx だけではなく aaaもあります． aaaについても解説しますか?」
- いちいち聞いてくるな必要ならばやれ

2. claude「aaa について参照することもできます」
- できるならやれ
